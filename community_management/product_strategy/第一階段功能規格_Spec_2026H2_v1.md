## 1. 資訊架構（IA）

### 1.1 頁面架構
- 第一層：`產品規劃主題`
- 第二層：`社區服務`

### 1.2 工作台區塊
1. `KPI 摘要列`：收繳率、催繳工時、結案時效、送達率。

5. `風險與依賴區`：高風險提醒、阻塞原因、下一步動作。

## 2. 互動規格（Interaction Spec）

### 2.1 流程切換（Tab）
- 元件語意：`role="tablist"` + `role="tab"` + `role="tabpanel"`。
- 預設值：首次進入預設 `催繳流程`。
- 狀態：active tab 需有主色樣式與 `aria-selected=true`。

### 2.2 任務卡片
- 每張卡片包含：標題、狀態 Badge、負責角色、SLA、下一步。
- 可操作按鈕：
  - `查看明細`（outline）
  - `標記已完成`（default）
- 鍵盤操作：可 Tab 聚焦按鈕；Enter 觸發。

### 2.3 週次里程碑
- 以垂直時間軸顯示（Now/Next 具差異化視覺）。
- 每個節點包含：週次、目標、完成定義（DoD）。

### 2.4 錯誤/空狀態
- 缺資料：顯示「缺漏欄位清單 + 下一步動作」。
- 無資料：顯示空狀態說明與「回到規劃」按鈕。

### 2.5 四模組詳細行為規格（補充）

#### 2.5.1 B-MVP 金流催繳看板
- **顯示內容：**
  - 逾期分群區塊（7/14/30 天）與優先催繳名單。
  - 催繳進度區塊（未聯繫/已聯繫/承諾繳款/已入帳）。
  - 對帳一致性區塊（已對帳/待確認/異常）。
- **互動行為：**
  - 點選任務卡可進入明細，更新聯繫狀態與承諾繳款日期。
  - 狀態更新後即時回寫 KPI 卡片與任務狀態 Badge。
- **例外處理：**
  - 缺欄位匯入時，顯示阻擋提示與缺漏清單。
  - 同戶重複資料時，顯示衝突提示與建議合併規則。

#### 2.5.2 A-MVP 高齡代辦工單
- **顯示內容：**
  - 工單卡片包含服務類型、責任人、SLA、狀態與證據完成度。
  - 顯示即將逾時與逾時工單分組。
- **互動行為：**
  - 建單時必填服務類型、SLA、責任人。
  - 結案前檢查履約證據，通過後觸發家屬通知。
  - 通知結果需顯示送達、失敗、重送中三種狀態。
- **例外處理：**
  - 派工不可用時允許改派，並保留改派軌跡。
  - 通知失敗時進入重送佇列，顯示錯誤原因與最後重送時間。

#### 2.5.3 F-Foundation 權限與稽核基線
- **顯示內容：**
  - 敏感操作前顯示授權檢查結果（允許/拒絕）。
  - 稽核檢視頁可查詢操作者、操作時間、操作資源、結果。
- **互動行為：**
  - 角色矩陣以最小權限為預設，超權限操作需阻擋。
  - 每次敏感操作成功或失敗都需寫入稽核日誌。
- **例外處理：**
  - 權限不足時顯示缺少權限名稱與申請流程提示。
  - 日誌寫入失敗時需顯示 P0 警示並禁止操作完成狀態。

#### 2.5.4 J-Lite 租客責任欄位
- **顯示內容：**
  - 案件卡片與明細頁顯示責任方（住戶/租客/物業）與覆核狀態。
  - 爭議案件列表可按責任方篩選。
- **互動行為：**
  - 責任方欄位可編輯，但每次異動都必須填寫變更原因。
  - 覆核通過後鎖定當前責任版本供後續追溯。
- **例外處理：**
  - 未填變更原因不得儲存責任異動。
  - 覆核中案件若再變更責任方，需顯示覆核重跑提示。

## 3. 視覺與 Design System 對齊

### 3.1 Foundations（沿用）
- Color：`bg-card`、`bg-muted/20`、`text-muted-foreground`、`text-primary`
- Typography：`h2` 主區塊、`h3` 子區塊、`text-sm` 內文、`text-xs` 輔助資訊
- Spacing：`p-4/p-6`、`gap-4/gap-6`
- Radius/Border：`rounded-xl/2xl`、`border`

### 3.2 Components（沿用）
- `Button`：`default`、`outline`、`ghost`
- `Status Badge`：`recorded / in-progress / next / p0 / p1 / p2`
- `Insight Card`：承載 KPI、風險、依賴、下一步

### 3.3 Patterns（沿用）
- `Dashboard Section Pattern`：Section Header -> Content Grid -> Summary Block
- `Report Pattern`：Filter Controls -> Metrics -> Cards -> Summary

## 4. 資料欄位規格（Data Contract）

### 4.1 KPI 指標
| 欄位 | 型別 | 範例 | 說明 |
|---|---|---|---|
| `id` | string | `collection-rate` | 指標識別 |
| `label` | string | `管理費收繳率` | 顯示名稱 |
| `value` | string | `81%` | 目前值 |
| `target` | string | `+8%` | 目標值或方向 |
| `status` | enum | `on-track` | `on-track` / `risk` / `blocked` |

### 4.2 任務卡
| 欄位 | 型別 | 範例 | 說明 |
|---|---|---|---|
| `id` | string | `b-reminder-batch` | 任務識別 |
| `track` | enum | `collection` | `collection` / `elder-care` |
| `title` | string | `逾期分群催繳批次` | 任務名稱 |
| `owner` | string | `社區管理員` | 負責角色 |
| `sla` | string | `24 小時內完成聯繫` | 時效要求 |
| `priority` | enum | `P0` | `P0` / `P1` / `P2` |
| `status` | enum | `in-progress` | `recorded` / `in-progress` / `next` |
| `nextAction` | string | `補齊 30 天逾期模板` | 下一步 |

### 4.3 里程碑
| 欄位 | 型別 | 範例 |
|---|---|---|
| `week` | string | `W3` |
| `goal` | string | `完成 Demo 與流程評審` |
| `definitionOfDone` | string | `通過 PM/Design/FE review` |
| `state` | enum | `in-progress` |

## 5. 驗收標準（Acceptance Criteria）

### AC-1 導覽與語意
- 進入第一階段 Demo 可在導覽中看見完整層級。
- 頁面標題與區塊標題層級連續（H2 -> H3）。

### AC-2 無障礙
- 流程切換可鍵盤操作且有正確 ARIA。
- 互動按鈕可聚焦且具 focus 樣式。

### AC-3 資訊層級
- 需同時呈現 KPI、任務、里程碑、風險四個層次。
- 不能只靠字級區分，需有卡片/邊框/間距節點。

### AC-4 與 PRD 對齊
- Demo 畫面需可對應 PRD 的 B/A/F/J-Lite 四模組語意。
- 每個流程至少有 2 個可操作示例任務。
