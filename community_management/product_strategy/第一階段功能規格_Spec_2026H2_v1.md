# 第一階段功能規格 Spec（社區服務）_2026H2

**一句話核心摘要：第一階段 Spec 定義管理端工作台的資訊架構、互動行為、資料欄位與驗收準則，確保 B/A/F/J-Lite 能在 12 週內可交付、可驗證、可擴充。**

**最後更新：** 2026-06-19  
**狀態：** 草稿（待 FE/BE/QA 評審）

---

## 1. 資訊架構（IA）

### 1.1 頁面架構
- 第一層：`產品規劃主題`
- 第二層：`社區服務`
- 第三層：`第一階段 Demo`（承接規劃）

### 1.2 工作台區塊
1. `KPI 摘要列`：收繳率、催繳工時、結案時效、送達率。
2. `流程切換列`：催繳流程 / 高齡代辦（Tab）。
3. `主要內容區`：依流程顯示清單卡片與狀態。
4. `週次執行區`：W1-W12 里程碑與當前週目標。
5. `風險與依賴區`：高風險提醒、阻塞原因、下一步動作。

## 2. 互動規格（Interaction Spec）

### 2.1 流程切換（Tab）
- 元件語意：`role="tablist"` + `role="tab"` + `role="tabpanel"`。
- 預設值：首次進入預設 `催繳流程`。
- 狀態：active tab 需有主色樣式與 `aria-selected=true`。

### 2.2 任務卡片
- 每張卡片包含：標題、狀態 Badge、負責角色、SLA、下一步。
- 可操作按鈕：
  - `查看明細`（outline）
  - `標記已完成`（default）
- 鍵盤操作：可 Tab 聚焦按鈕；Enter 觸發。

### 2.3 週次里程碑
- 以垂直時間軸顯示（Now/Next 具差異化視覺）。
- 每個節點包含：週次、目標、完成定義（DoD）。

### 2.4 錯誤/空狀態
- 缺資料：顯示「缺漏欄位清單 + 下一步動作」。
- 無資料：顯示空狀態說明與「回到規劃」按鈕。

## 3. 視覺與 Design System 對齊

### 3.1 Foundations（沿用）
- Color：`bg-card`、`bg-muted/20`、`text-muted-foreground`、`text-primary`
- Typography：`h2` 主區塊、`h3` 子區塊、`text-sm` 內文、`text-xs` 輔助資訊
- Spacing：`p-4/p-6`、`gap-4/gap-6`
- Radius/Border：`rounded-xl/2xl`、`border`

### 3.2 Components（沿用）
- `Button`：`default`、`outline`、`ghost`
- `Status Badge`：`recorded / in-progress / next / p0 / p1 / p2`
- `Insight Card`：承載 KPI、風險、依賴、下一步

### 3.3 Patterns（沿用）
- `Dashboard Section Pattern`：Section Header -> Content Grid -> Summary Block
- `Report Pattern`：Filter Controls -> Metrics -> Cards -> Summary

## 4. 資料欄位規格（Data Contract）

### 4.1 KPI 指標
| 欄位 | 型別 | 範例 | 說明 |
|---|---|---|---|
| `id` | string | `collection-rate` | 指標識別 |

| `status` | enum | `on-track` | `on-track` / `risk` / `blocked` |

### 4.2 任務卡

|---|---|---|---|

| `title` | string | `逾期分群催繳批次` | 任務名稱 |
| `owner` | string | `社區管理員` | 負責角色 |

| `status` | enum | `in-progress` | `recorded` / `in-progress` / `next` |

### 4.3 里程碑
| 欄位 | 型別 | 範例 |

| `state` | enum | `in-progress` |

## 5. 驗收標準（Acceptance Criteria）

### AC-1 導覽與語意
- 進入第一階段 Demo 可在導覽中看見完整層級。
- 頁面標題與區塊標題層級連續（H2 -> H3）。

### AC-2 無障礙
- 流程切換可鍵盤操作且有正確 ARIA。
- 互動按鈕可聚焦且具 focus 樣式。

### AC-3 資訊層級
- 需同時呈現 KPI、任務、里程碑、風險四個層次。
- 不能只靠字級區分，需有卡片/邊框/間距節點。

### AC-4 與 PRD 對齊
- Demo 畫面需可對應 PRD 的 B/A/F/J-Lite 四模組語意。
- 每個流程至少有 2 個可操作示例任務。
